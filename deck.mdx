<Image src="static/meme.jpg" width="auto" height="600px" />

---

### What?

<ul>
  <Appear>
    <li>
        computation as the evaluation of mathematical functions
    </li>
    <li>
        expressions or declarations instead of statements
    </li>
  </Appear>
</ul>

---

### How?

<ul>
  <Appear>
    <li>
        <b>Statelessness</b> - an operation is not sensitive to the state of the computation
    </li>
    <li>
        <b>Referential transparency</b> - an expression can be replaced with its corresponding value without changing the
        program's behavior
    </li>
    <li>
        <b>Immutability</b> - an object whose state cannot be modified after it is created
    </li>
  </Appear>
</ul>

---

### Why?

<ul>
  <Appear>
    <li>
        more predictable code
    </li>
    <li>
        less verbose code
    </li>
    <li>
        less error-prone
    </li>
    <li>
        easier to test
    </li>
    <li>
        easily reusable code
    </li>
    <li>
        faster to write code
    </li>
  </Appear>
</ul>

### What it isn't

<ul>
  <Appear>
    <li>
      <b>Loops</b> (while, do ... while, for, for ... of, for ... in)
    </li>
    <li>
      <b>Mutable variable declarations</b> (var or let)
    </li>
    <li>
      <b>Void functions</b>
    </li>
    <li>
      <b>Object mutations</b>
    </li>
    <li>
      <b>Array mutator methods</b> (copyWithin, fill, pop, push, reverse, shift, sort, splice, unshift)
    </li>
    <li>
      <b>Map mutator methods</b> (clear, delete, set)
    </li>
    <li>
      <b>Set mutator methods</b> (add, clear, delete)
    </li>
  </Appear>
</ul>

---

### More concepts

**Function composition** - mathematical concept that allows you to combine two or more functions into a new function

**Tactic programming / point-free style** - a programming paradigm in which function definitions do not identify the
arguments (or "points") on which they operate

<Split>
<div>

```js
const increment = x => x + 1;
const double = x => x * 2;
const incrementAndDouble = (x) => increment(double(x));
// or: const incrementAndDouble = compose(increment, double);
// or: const incrementAndDouble = pipe(double, increment);

const result = incrementAndDouble(2);
```

</div>
<div>

```js
const array = [1, 2, 3];
const result = array.map(x => incrementAndDouble(x));
const pointFreeResult = array.map(incrementAndDouble);
```

```js
const snakeCase = word => word.toLowerCase().replace(/\s+/ig, '_');
const snakeCase = compose(replace(/\s+/ig, '_'), toLowerCase);
```

</div>
</Split>

### More concepts

**Currying** - translating the evaluation of a function that takes multiple arguments into evaluating a sequence of
functions, each with a single argument

**Partial application** - applying some (but not all) of the arguments to a function and returning a new function
awaiting the rest of the arguments


<Split>
<div>

```js
const add = (x, y) => x + y;
const result = add(2, 3);

const curriedAdd = (x) => (y) => x + y;
// or: const curriedAdd = curry(add);
const curriedResult = curriedAdd(2)(3);
```

</div>
<div>

```js
const add2 = (x) => add(x, 2);
const result = add2(3);

const partialAdd2 = curriedAdd(2);
// or: const partialAdd2 = partial(add, 2);
const partialResult = partialAdd2(3);
```

</div>
</Split>

### More concepts

<Split>
<div>

Normal people
```js
const array = [1, 2, 3];
const multiply = (x, y) => x * y;
const doubledArray = array.map((x) => multiply(x, 2));
const tripledArray = array.map((x) => multiply(x, 3));
```

</div>
<div>

Functional people
```js
const array = [1, 2, 3];
const multiply = (x) => (y) => x * y;
const doubledArray = array.map(multiply(2));
const tripledArray = array.map(multiply(3));
```

</div>
